WTF is node.js ?

The illegitimate child of Google and Netscape !

2008, Ryan Dahl

How to build scalable real-time web app with simplicity ?
- 2008: Ryan Dahl needed a new way to build _real-time_ web app (push based), with 2 priorities: scalability and simplicity. - Netscape -> JS -> JavaScript the good parts - Gmail -> Chome -> V8 - Non blocking I/O for scalability - JS for simplicity

Architecture

  • Non blocking I/O
  • Event driven model
  • JavaScript
  • Unix philosophy

Inspiration

  • Twisted (Python)
  • Tornado (Python)
  • EventMachine (Ruby)
  • libevent (C)
  • Nginx

Non blocking I/O

for scalability

Threaded server

Non blocking server


nginx stops scalling if he calls a blocking app.
Node is designed to be non blocking from top to bottom.
  • Event driven model
  • Rewrite all low level I/O APIs
  • Fat process
- Only one fat process (no threads, etc.) (yes but how to scale?) - No blocking I/O: the unique process share the memory and can run something else by waiting the end of the I/O - Event driven model (main loop, then event execution sequentially)

JavaScript

for simplicity

Why the choice of JS?

Ryan Dahl would do it in C (using the libevent) but changes his mind when he read the book of crockford, and saw the performances of V8

The good parts for node

  • Functionnal language : fits the event model
  • Easily extensible : redesign of non blocking APIs, DSLable
  • Becomes popular : the language of the web
- Functionnal language : callbacks, event loop and event driven programming feets well the everything must be non blocking approach - Interpreted language : simple to use and to extends - JS becomes popular

Global State
